Game explanation :

The game of Web speed is based on fast-thinking and self-improvement through playing, at your disposal are a set of templates (from beginners to advanced challenges) and 2 deck of cards for each team (HTML/CSS).

This game shall teach you basic HTML/CSS architectures and ways of organizing code.

Teams are made-up of two people one in charge of the CSS deck and the other in charge of the HTML deck, the goal is to analyze and recreate the template that will be shown to all teams.

Course of the game :

After a template card was selected the two teams can begin analysing what they will have to put and where and also organizing themselves to begin the puzzle, the player in charge of the HTML cards has to put down on their blank template sheet the different HTML tags and the player in charge of the CSS must place the right style on each HTML card.

The first team to successfully complete the template card wins a round, first team to five rounds wins the game.

If upon inspection by the other teams the first team to finish the template has an error in there architecture, everybody except said team wins the round.

Rules :

The game of Web Speed is pretty well framed so not much rules are needed however, remember those

  • All of the teams shall start building there website at the same moment
  • CSS cards can be put on top of eachother
  • You can only put CSS card on pre-existing places (on the HTML cards that allows it)
  • Do not try to stop any other teams and just focus on building your website the fastest way possible
  • You are assigned a single deck of card, you can'y help your teammate by placing his cards
  • After someone says out loud "DONE !" all teams must stop building and begin verification of the team that declared themselves over with the building.
  • Players will ignore the border styles and shadow styles on the templates

About the cards

The HTML deck :

  • div = Basic HTML container, easily maleable hence it's popularity
  • header = an HTML element that represents introductory content, usually navigational aids.
  • footer = an HTML element that represents the last ontainer of a web page, usually navigational aids
  • h1 = an HTML heading, that serves as a title holder (h2, h3, h4, etc... also exists)
  • p = a paragraph container, used to display text onto the web page
  • ul = an HTML container meant to display list items
  • li = HTML element that is an item within a list

The CSS deck :

  • background-color = changes the background color of the container on wich it stands
  • color = changes the color of the text on wich it stands
  • font-family = changes the font family (arial, open sans, etc) of the text on wich it stands
  • text-transform = CSS property that allow to transform text in many ways

about custom selectors = you maybe have stumbled upon a card using a ">p" or ">h1" before the css content,, it is a way of telling the navigator to apply the style to the first paragraph or heading1 they see within the container on wich the style is placed
